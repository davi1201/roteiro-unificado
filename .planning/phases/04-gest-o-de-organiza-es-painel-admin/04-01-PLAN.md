## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → React Router | URL e parâmetros (orgId) entram no app via roteamento; podem ser manipulados pelo usuário |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | AdminRoute → AdminLayout | mitigate | AdminLayout NUNCA é renderizado fora do `<AdminRoute />` no router.tsx — guard upstream valida `role === 'admin'` e bloqueia outros papéis. Verificação adicional não é necessária no layout (defense-in-depth via RLS em queries futuras). |
| T-04-02 | Information Disclosure | AdminHeader exibe `user.email` | accept | Email do próprio admin logado não é informação sensível neste contexto; nenhuma org/usuário externo é exposto. |
| T-04-03 | Tampering | Dialog backdrop bypass via click no panel | mitigate | Panel usa `onClick={(e) => e.stopPropagation()}` para evitar que clicks dentro do panel borbulhem ao overlay; apenas clicks no overlay disparam `onClose`. |
| T-04-04 | Denial of Service | Body scroll lock não restaurado se Dialog desmontar abruptamente | mitigate | `useEffect` cleanup function restaura `document.body.style.overflow = ''` mesmo em desmontagem forçada (React garante cleanup). |
| T-04-05 | Spoofing | URL `/admin/dashboard` acessada via deep link sem auth | mitigate | AdminRoute upstream redireciona para `/login` quando `session === null` ou `role !== 'admin'`. Comportamento existente — não alterar. |
