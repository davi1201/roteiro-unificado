<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI / Dashboard → Supabase remote | Migrations executam com privilégio de superuser no banco remoto |
| seed.sql passwords | Hash via pgcrypto no momento de execução — senhas plaintext nunca persistidas no banco |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-D01 | Tampering | supabase db reset no remote | mitigate | Comentário explícito em seed.sql e neste plano proibindo db reset no remote; usar apenas db push |
| T-02-D02 | Information Disclosure | SUPABASE_ACCESS_TOKEN em env | mitigate | Token de acesso não deve ser commitado em .env; usar apenas em env var local temporária ou via supabase login browser flow |
| T-02-D03 | Tampering | Migration aplicada fora de ordem | mitigate | Prefixos de timestamp garantem ordem correta; dry-run confirma sequência antes do push real |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Este plano não instala pacotes npm; Supabase CLI instalado via brew (tap oficial) no Plano 01 |
</threat_model>
