<objective>
Implementar os dois hooks de mutation que completam o fluxo de versionamento append-only:
(1) `useSubmitAssessment(orgId)` — faz o draft virar imutável via UPDATE status='submitted'; 
(2) `useNewRevision(orgId)` — cria nova revisão draft copiando form_data da versão submitted mais
recente via INSERT, preservando o histórico intocado.

Estes hooks seguem o padrão exato de useArchiveOrg.ts (analog confirmado no 08-PATTERNS.md):
useMutation com mutationFn que throw em erro, onSuccess que invalida queries + toast + navigate,
onError com toast de erro.

Purpose: Completar o ciclo de vida de uma avaliação — draft → submitted → nova revisão.
Output: Dois arquivos de hook em src/features/form/ prontos para uso no FormLayout e HistoryPage.
</objective>

## Trust Boundaries

| Boundary | Descrição |
|----------|-----------|
| Frontend mutation → assessments | UPDATE e INSERT em dados do banco via Supabase client autenticado |
| version increment | Incremento client-side baseado em SELECT prévio — risco teórico de race condition |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-08 | Tampering | useSubmitAssessment — UPDATE em submitted | mitigate | `.eq('status', 'draft')` no WHERE garante que apenas drafts são transicionados; RLS da Phase 2 (assessments_update_draft) bloqueia qualquer UPDATE em submitted no banco |
| T-08-09 | Tampering | useNewRevision — INSERT copiando form_data | mitigate | INSERT com org_id explícito do parâmetro orgId; org_id não vem de user input; RLS assessments_insert valida is_org_member(org_id) |
| T-08-10 | Elevation of Privilege | version increment com race condition | accept | Risco aceito para piloto com 5 construtoras; implementação two-step (SELECT + UPDATE) documentada; tech debt para RPC atômica em escala |
| T-08-11 | Denial of Service | INSERT falhando por draft já existente | mitigate | A constraint UNIQUE parcial da migration 08-01 bloqueia segundo INSERT; erro propagado via throw para o onError do useMutation |
| T-08-SC | Tampering | npm installs | accept | Nenhum pacote novo instalado neste plano |
