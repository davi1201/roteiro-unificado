    Nome: 20260523000001_assessments_draft_unique.sql
    Descrição: Phase 8 — adiciona constraint UNIQUE parcial para garantir no máximo um draft ativo por org.

    A migration deve:
    1. Criar um índice único parcial via CREATE UNIQUE INDEX: nome sugerido
       `assessments_org_id_draft_unique`, em `public.assessments (org_id)` com
       `WHERE (status = 'draft')`.
       Usar CREATE UNIQUE INDEX porque PostgreSQL suporta índices únicos parciais (WHERE clause),
       e o Supabase upsert com `onConflict` funciona com índices únicos, não apenas com
       constraints UNIQUE declaradas no DDL.
    2. Adicionar comentário explicando por que é parcial: múltiplos registros submitted por org
       são válidos (append-only); apenas um draft por org deve existir por vez.
    3. Usar `IF NOT EXISTS` para tornar a migration idempotente.

    Após criar o arquivo, executar `supabase db push` no diretório raiz do projeto
    (onde supabase/config.toml está localizado) para aplicar ao banco remoto.

    Se o `supabase db push` falhar por conflito de dados (ex: existem dois drafts para a mesma org),
    o executor deve inspecionar os dados com `supabase db remote` ou listar os assessments duplicados
    e remover os duplicados antes de re-executar o push. Não silenciar o erro.
