## Diagnóstico do Bug (SAVE-03)

A política `assessments_update_draft` em `20260522000008` tinha:

```sql
WITH CHECK (
  (SELECT public.is_org_member(org_id))
  AND status = 'draft'   -- avalia a NOVA linha após UPDATE
)
```

PostgreSQL avalia `WITH CHECK` contra a **nova linha** resultante do UPDATE. Quando `useSubmitAssessment` executa `UPDATE SET status='submitted'`, a nova linha tem `status='submitted'`, violando o CHECK — erro RLS 42501.

## Fix Aplicado

**Arquivo criado:** `supabase/migrations/20260523000002_fix_rls_assessments_submit.sql`

```sql
DROP POLICY IF EXISTS "assessments_update_draft" ON public.assessments;

CREATE POLICY "assessments_update_draft"
  ON public.assessments FOR UPDATE
  TO authenticated
  USING (
    (SELECT public.is_org_member(org_id))
    AND status = 'draft'                         -- inalterado: só drafts são visíveis para UPDATE
  )
  WITH CHECK (
    (SELECT public.is_org_member(org_id))
    AND status IN ('draft', 'submitted')         -- ÚNICO change: aceita nova linha como submitted
  );
```

## Verificação do Banco (Task 2)

Query `pg_policies` retornou:

```json
{
  "policyname": "assessments_update_draft",
  "cmd": "UPDATE",
  "qual": "(( SELECT is_org_member(assessments.org_id)) AND (status = 'draft'::assessment_status))",
  "with_check": "(( SELECT is_org_member(assessments.org_id)) AND (status = ANY (ARRAY['draft'::assessment_status, 'submitted'::assessment_status])))"
}
```

- `qual` (USING): contém apenas `status = 'draft'` — regressão submitted→draft permanece bloqueada ✓
- `with_check`: contém `status = ANY (ARRAY['draft', 'submitted'])` — submissão permitida ✓
