<objective>
Corrigir o bug de RLS que bloqueia a submissão de avaliações (SAVE-03). A política `assessments_update_draft` foi escrita com `WITH CHECK (status = 'draft')`, e o PostgreSQL avalia o `WITH CHECK` contra a NOVA linha resultante do UPDATE. Quando `useSubmitAssessment` executa `UPDATE SET status = 'submitted'`, a nova linha viola o CHECK e o PostgreSQL retorna erro RLS 42501 — a submissão cai no `onError` do hook.

O fix é cirúrgico: uma migration nova dropa e recria a policy, mantendo `USING (status = 'draft')` (só drafts podem ser atualizados, bloqueando regressão submitted→draft) e ampliando `WITH CHECK` para `status IN ('draft', 'submitted')`. Nenhum código TypeScript muda — o hook `useSubmitAssessment.ts` já está correto.

Purpose: Desbloquear SAVE-03, o único blocker crítico do audit v1.0, para que construtoras possam submeter avaliações.
Output: 1 migration SQL nova aplicada ao banco remoto + policy verificada via pg_policies + confirmação end-to-end no browser.
</objective>
