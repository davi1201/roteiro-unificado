## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Usuário (browser) → React state | Inputs do usuário entram via Controller/onChange dos field components; permanecem em memória/Zustand até Phase 8 sincronizar com Supabase. Nenhum boundary atravessado por esta fase. |
| ConditionalField → RHF internals | Chamada manual de `unregisterFn` precisa ser feita apenas em condições conhecidas pelo Section pai. Risco: chamadas espúrias podem apagar dados não esperados. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Tampering | RadioGroupField / CheckboxGroupField — botões dentro de `<form>` futuro | mitigate | Todos os `<button>` em RadioGroupField usam `type="button"` explicitamente para nunca disparar submit acidental do Section form. Verificado via grep no acceptance_criteria. |
| T-06-01-02 | Tampering | CheckboxGroupField — manipulação de array | mitigate | Controller único gerencia o array inteiro; cada onChange recalcula o array a partir do `field.value` atual, evitando race conditions entre múltiplos toggles. Dependência única no Controller render. |
| T-06-01-03 | Information Disclosure | ConditionalField — unregister com keepValue:false | accept | Comportamento intencional: ao ocultar campo condicional, valor é removido da store do RHF para evitar submissão de campo invisível com valor stale. Aceito por design (FORM-06 + RESEARCH §Armadilha 3). Sem PII em jogo nesta camada. |
| T-06-01-04 | Tampering | Field components — XSS via `field.value` injetado em label/option | mitigate | Field components renderizam `field.value` apenas em estado interno do Controller; labels vêm de `options[].label` definidos em código (não vindos do usuário). React escapa automaticamente. Sem `dangerouslySetInnerHTML` em nenhum lugar. |
| T-06-01-SC | Tampering | Pacotes npm | accept | Nenhum pacote novo é instalado nesta fase (RESEARCH §Auditoria de Pacotes confirma). `react-hook-form@7.76.0` e `zod@4.4.3` já auditados nas fases anteriores. |
