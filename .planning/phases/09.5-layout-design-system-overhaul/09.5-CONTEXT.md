# Phase 9.5: Layout & Design System Overhaul — Context

**Gathered:** 2026-05-24
**Status:** Ready for planning
**Source:** Sketch findings (sessions 001-sidebar-layout + 002-form-inputs, 2026-05-24)

<domain>
## Phase Boundary

Refactor visual dos shells admin e formulário com base nas decisões do sketch. Sem nova funcionalidade de negócio — apenas melhoria de UX/layout. Não altera rotas, autenticação, lógica de autosave, schemas Zod ou stores.

**Entrega principal:**
1. Sidebar admin colapsável (240px ↔ 60px) com toggle + topbar branca com breadcrumb
2. Formulário de avaliação com abas organizadas em cards semânticos + footer sticky prev/next + indicador de autosave

</domain>

<decisions>
## Implementation Decisions

### Sidebar Admin (AdminSidebar.tsx)

- **Colapso via estado React** `useState<boolean>(collapsed)` — persiste em `localStorage` com key `sidebar_collapsed`
- **Largura:** 240px expandida, 60px colapsada — via CSS `width` com `transition: width 0.25s ease`
- **Toggle button:** `position: absolute; top: 12px; right: -12px`, diâmetro 24px, fundo branco, borda cinza, shadow-sm; ícone ChevronLeft/ChevronRight 14px
- **Nav items:** `display:flex; gap:10px; padding:10px; border-radius:6px; font-size:13.5px; white-space:nowrap; overflow:hidden; transition:background 0.15s`
  - Ativo: `background: rgba(255,255,255,0.15); font-weight:600; color:white`
  - Hover: `background: rgba(255,255,255,0.08); color:white`
  - Desabilitado: `opacity:0.4; cursor:not-allowed`
- **Labels:** `display:none` quando `collapsed=true` (no CSS: `.sidebar.collapsed .nav-label { display:none }`)
- **User pill** no `<footer>` da sidebar: avatar circular com inicial, email truncado, botão "Sair" — substitui o AdminHeader para sign-out
- **Ícones:** SVG inline 18×18px, `strokeWidth={1.5}`, existentes preservados

### AdminHeader.tsx → Topbar branca com breadcrumb

- **Background:** `bg-white border-b border-gray-200 shadow-[0_1px_3px_rgba(0,0,0,0.05)]` (não mais `bg-primary`)
- **Position:** `sticky top-0 z-20 h-14 flex items-center justify-between px-6`
- **Breadcrumb esquerdo:** `<nav>` com `font-size:13px; color:#6b7280` + separador `›` + último item `font-weight:600; color:#111827`
- **Breadcrumb dinâmico:** recebe `breadcrumb: Array<{label: string, href?: string}>` como prop
- **Direita:** slot para ações CTA (ex: botão "Nova Organização") — via prop `actions?: ReactNode`
- **Sign-out** migrado para user pill na sidebar — AdminHeader não tem mais botão de logout

### AdminLayout.tsx

- **Main area:** `margin-left` dinâmico — 240px expandida, 60px colapsada, com `transition: margin-left 0.25s ease`
- **Estado collapsed** compartilhado: `useSidebarState()` hook (`useState` + localStorage) importado em AdminSidebar e AdminLayout
- **AdminHeader** recebe `breadcrumb` e `actions` derivados da rota atual (via `useMatch` / `useLocation`)

### FormLayout.tsx — topbar do formulário

- **Strip de status no topo** (não é sidebar): faixa `bg-white border-b border-gray-200 h-10 flex items-center px-4 justify-between`
  - Esquerda: nome da aba ativa (`font-size:13px; font-weight:600`)
  - Direita: `<AutosaveIndicator>` — ponto verde 6px + "Salvo há X min" em 11px
- **AutosaveIndicator:** novo componente `src/features/form/AutosaveIndicator.tsx` — recebe `lastSaved: Date | null` como prop; exibe "Salvando…" durante debounce, "Salvo há X min" depois, "Falha ao salvar" em erro; usa `useAutosave` internamente se necessário

### FormLayout.tsx — footer sticky em todas as abas

- **Footer universal** (hoje só aparece na aba NDA): `position:sticky; bottom:0; bg-white; border-t; padding:12px 24px`
- **Layout:** `justify-between`
  - Esquerda: `<Button variant="ghost">← Anterior</Button>` (desabilitado na primeira aba)
  - Centro: `<Button variant="ghost" size="sm">Salvar rascunho</Button>` (link-style)
  - Direita: `<Button variant="primary">Próxima aba →</Button>` (ou "Enviar Avaliação" na aba NDA)
- **Navegação:** `prev/next` baseados no `TAB_CONFIG` — `store.setActiveTab(prevKey)` + update hash
- **Botão Enviar** permanece apenas na aba NDA, substitui "Próxima aba →" no footer

### Form sections — card layout semântico

- **Wrapper por grupo:** cada grupo semântico de campos dentro de uma Section vira um `<FormCard>` com header colorido
- **Componente `FormCard`:** `src/features/form/FormCard.tsx`
  - Props: `icon: ReactNode`, `iconColor: 'blue'|'amber'|'green'|'purple'`, `title: string`, `subtitle?: string`, `children: ReactNode`
  - Header: `padding:11px 16px; background:#f8fafc; border-bottom:1px solid #e5e7eb; display:flex; gap:10px; align-items:center`
  - Card-icon: `width:28px; height:28px; border-radius:6px; display:flex; align-items:center; justify-content:center; color:white`
  - Cores: blue=`#1e3a5f`, amber=`#f59e0b`, green=`#22c55e`, purple=`#7c3aed`
  - Body: `padding:16px`
- **Grid de cards:** `<FormCardRow cols={1|2}>` — `display:grid; grid-template-columns:1fr` ou `1fr 1fr; gap:16px`
- **Regra cols-2:** usar quando dois grupos têm densidade similar, são complementares, nenhum tem textarea longa
- **Prioridade de implementação:** IdentificacaoSection primeiro (prova de conceito), depois restantes em Wave paralela

### CSS / Tailwind tokens

- Token `--color-primary` PERMANECE `#123B66` (cor do projeto, não mudar para `#1e3a5f` do sketch — o sketch usou exemplo genérico)
- Token `--color-accent` PERMANECE `#F28C28`
- Adicionar tokens: `--sidebar-width:240px`, `--sidebar-width-collapsed:60px`, `--header-height:56px`
- `--font-sans: 'Inter', system-ui, sans-serif` — adicionar import Google Fonts no `index.html`
- Focus ring padrão para todos os inputs: `box-shadow: 0 0 0 3px rgba(37,99,235,0.1)`
- Validação visual blur: borda verde (`#22c55e`) = ok, borda vermelha (`#ef4444`) = erro

### Claude's Discretion

- Ordem exata de implementação dos card groups dentro de cada section (exceto Identificação)
- Ícones SVG específicos para cada card-icon (desde que sejam monocromo inline 14-16px)
- Breakpoint exato para colapso mobile da sidebar do formulário
- Número exato de planos e waves

</decisions>

<specifics>
## Specific Ideas

- Sidebar do sketch (001B) usa `position:fixed` — manter fixed para não rolar com conteúdo
- FormCard deve ser presentacional puro — sem lógica de store, só styling
- AutosaveIndicator deve ser leve — observa `lastSavedAt` do store via selector, sem polling
- Footer do formulário SUBSTITUIU o sticky footer exclusivo da aba NDA — agora universal; botão "Enviar" aparece na aba NDA no lugar de "Próxima aba →"
- Breadcrumb do AdminHeader: página admin principal = "Admin › Organizações", detalhe de org = "Admin › Organizações › {nomeOrg}"

</specifics>

<deferred>
## Deferred Ideas

- Dark mode — não é requisito do piloto
- Animações além de transições 0.25s (ex: Framer Motion) — não necessário para o piloto
- Exportação de PDF/Excel dentro desta fase — escopo das Fases 10 e 11
