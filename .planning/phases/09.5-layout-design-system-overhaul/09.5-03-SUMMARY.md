# Phase 09.5 Plan 03: Footer Universal + Topbar de Status no FormLayout

**One-liner:** Campo `lastSavedAt` ephemeral no formStore + AutosaveIndicator wired no topbar do FormLayout + footer sticky universal com navegação prev/next e botão "Enviar Avaliação" exclusivo da aba NDA.

## O Que Foi Construído

### Task 1: lastSavedAt no formStore + setLastSaved no useAutosave (TDD)

**`formStore.ts`**: Adicionado campo `lastSavedAt: Date | null` ao `FormState` (default `null`) e action `setLastSaved(date: Date | null)` ao `FormActions`. Campo é ephemeral — excluído do `partialize` para não persistir em localStorage, e resetado para `null` em `onRehydrateStorage` (defensivo) e em `resetForm`.

**`useAutosave.ts`**: Após save bem-sucedido (branch `else` do `if (saveError)`), chama `store.getState().setLastSaved(new Date())` antes do toast. No branch de erro, `setLastSaved` não é chamado — `lastSavedAt` permanece inalterado.

**Testes adicionados:**
- `formStore.test.ts`: 4 testes no describe `lastSavedAt` — default null, setLastSaved atualiza, reset para null, não persiste em localStorage.
- `useAutosave.test.ts`: 3 testes em `useAutosave — lastSavedAt` — LAST-SAVED-01 (INSERT sucesso), LAST-SAVED-02 (UPDATE sucesso), LAST-SAVED-03 (erro não atualiza).

### Task 2: Topbar de status no FormLayout

Inserida strip de status (`h-10`, `border-b border-gray-200`) no topo do `<main>`, acima do conteúdo. Strip usa `-mx-4 mb-4 md:-mx-6` para contrabalançar o padding do `<main>` e ocupar largura total. Label da aba ativa à esquerda (`text-[13px] font-semibold`), `AutosaveIndicator` wired com `store.lastSavedAt` à direita. `<h1>` original removido — label migrou para a strip. `pb-20` condicional da NDA substituído por `pb-24` universal.

### Task 3: Footer universal prev/next/submit

Substituído o bloco condicional `{store.activeTab === TabKey.Nda && !draftQuery.isLoading && ...}` por footer universal renderizado em todas as 10 abas.

**Navegação derivada:**
```typescript
const activeIndex = TAB_CONFIG.findIndex((t) => t.key === store.activeTab)
const prevTab = activeIndex > 0 ? TAB_CONFIG[activeIndex - 1] : null
const nextTab = activeIndex >= 0 && activeIndex < TAB_CONFIG.length - 1 ? TAB_CONFIG[activeIndex + 1] : null
const isNdaTab = store.activeTab === TabKey.Nda

function goToTab(key: TabKey) {
  store.setActiveTab(key)
  window.history.replaceState(null, '', `#${key}`)
  store.markTabVisited(key)
}
```

**Layout do footer:**
- Esquerda: botão "← Anterior" (disabled quando `!prevTab` — aba Identificação)
- Centro-direita: botão "Salvar rascunho" (reassurance visual) + botão "Próxima aba →" (ou "Enviar Avaliação" na aba NDA)
- "Enviar Avaliação" abre `setIsSubmitOpen(true)` — dialog de submissão preservado intacto
