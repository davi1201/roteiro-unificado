# Phase 09.5 Plan 01: Primitivos de Design System — Tokens CSS, Hook Sidebar, FormCard, AutosaveIndicator

**One-liner:** Tokens Tailwind v4 de layout (sidebar/header) + fonte Inter + 4 primitivos reutilizáveis (hook `useSidebarCollapsed`, componentes `FormCard`, `FormCardRow`, `AutosaveIndicator`) que servem de base para as Waves 2 e 3 da Fase 9.5.

## O Que Foi Construído

### Task 1: Tokens de layout + fonte Inter

Adicionados 3 tokens ao `@theme {}` do `index.css` sem remover nenhum token existente:
- `--sidebar-width: 240px` → classe `w-sidebar`
- `--sidebar-width-collapsed: 60px` → classe `w-sidebar-collapsed`
- `--header-height: 56px` → classe `h-header`

Importação do Google Fonts Inter (400/500/600/700) adicionada no `<head>` do `index.html` antes do `<title>` para evitar FOUT.

### Task 2: Hook useSidebarCollapsed + AutosaveIndicator (TDD)

**`useSidebarCollapsed`**: Hook que retorna `[collapsed: boolean, toggle: () => void]`. Persiste em `localStorage` com key `sidebar_collapsed`. Inclui `try/catch` para proteção contra quota excedido e Safari private mode (mitiga STRIDE T-09.5-02).

**`AutosaveIndicator`**: Componente presentacional puro com 4 estados visuais:
- `lastSaved=null, isSaving=false` → "Não salvo"
- `isSaving=true` → "Salvando..."
- `lastSaved=Date, isSaving=false` → ponto verde (`bg-g5`) + "Salvo há X min" (com formatação em min/h)
- `error=true` → "Falha ao salvar" (texto `text-g1` — vermelho)

### Task 3: FormCard + FormCardRow (TDD)

**`FormCard`**: Card com header `bg-gray-50` + ícone colorido + título + subtítulo opcional + body. Mapping de cores por object lookup:
- `blue` → `bg-primary` (#123B66)
- `amber` → `bg-accent` (#F28C28)
- `green` → `bg-g5`
- `purple` → `bg-[#7c3aed]`

**`FormCardRow`**: Wrapper de grid responsivo. `cols=1` → `grid-cols-1`. `cols=2` → `grid-cols-1 md:grid-cols-2` (empilha em mobile, lado-a-lado em ≥ 768px).
