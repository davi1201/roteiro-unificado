<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Zustand store | Estado é client-side; dados em localStorage/sessionStorage são acessíveis por qualquer script da mesma origem |
| persist key namespace | `form-progress-${tenantId}` e `form-data-${tenantId}` — tenantId vem de `useParams().orgId` (rota URL controlada pelo usuário) |
| sectionData → sessionStorage | Phase 5 não persiste dados sensíveis (sem campos reais ainda); dados temporários só sobrevivem na sessão |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | sectionData persistindo em sessionStorage entre abas | mitigate | sessionStorage é mesma origem e isolada por tab — não acessível por outras abas/origens; chave namespaceada por tenantId; em Phase 8 Supabase será a fonte autoritativa, sessionStorage só rascunho local |
| T-05-02 | Tampering | Usuário manipula chave localStorage form-progress-* para outro tenantId e tenta carregar dados de outra org | accept | Phase 5 não há dados reais; persist key namespaceada limita escopo; RLS no Supabase (Phase 2) é a barreira autoritativa; defense-in-depth com routeOrgId === authOrgId fica no Plan 04 |
| T-05-03 | Tampering | sectionData vaza para localStorage por engano de partialize | mitigate | `partialize` retorna EXPLICITAMENTE só { currentStep, completedSteps, activeTab, visitedTabs } — sectionData omitido; acceptance criterion verifica ausência da string `sectionData` no bloco partialize via grep |
| T-05-04 | Denial of Service | sessionStorage quota exceeded ao persistir sectionData grande | accept | Phase 5 não há campos reais — payload é {} para todas as abas; subscriber envolve setItem em try/catch para ignorar quota errors silenciosamente; Phase 6 reavaliar se campos crescerem |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | Nenhum pacote novo é instalado nesta fase (per RESEARCH.md §Stack Padrão) — não há legitimidade a verificar |
</threat_model>
