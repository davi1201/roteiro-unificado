# Quick Task 260524-n1y: main do FormLayout ocupa 100% do espaço

## Contexto

Problema: inner div (aside+main container) tem `flex-1` no root `flex-col min-h-screen` = altura de `100vh - 4px` (ProgressBar h-1 sticky ocupa 4px no flow). Aside tem `md:h-screen` = `100vh`. Divergência: aside é 4px mais alto que o container. Main não preenche altura total.

Referência: AdminLayout usa `min-h-screen` no content wrapper div (não `flex-1`), garantindo que a área de conteúdo sempre cubra pelo menos 100vh.

## Tarefa

### T1: Aplicar mesma lógica do AdminLayout no FormLayout

**Arquivo:** `roteiro-unificado/src/features/form/FormLayout.tsx`

**Mudança:** linha 179 — trocar `flex-1` por `min-h-screen` no inner wrapper div:

```diff
- <div className="flex flex-1 flex-col md:flex-row">
+ <div className="flex min-h-screen flex-col md:flex-row">
```

**Resultado:** Inner div sempre cobre pelo menos 100vh (igual ao aside `h-screen`). `align-items: stretch` (padrão flex) faz o `main` esticar até a altura do container. `flex-1` no main preenche a largura restante no desktop.

**Commit:** `fix(260524-n1y): make FormLayout main fill full height following AdminLayout pattern`
