## Critical Blocker — SAVE-03

### RLS Policy Prevents Assessment Submission

**File:** `supabase/migrations/20260522000008_rls_policies_assessments.sql` lines 38–48

The `assessments_update_draft` policy `WITH CHECK` clause requires `status = 'draft'` on the **resulting row**. When `useSubmitAssessment` updates `{ status: 'submitted' }`, the resulting row has `status = 'submitted'` — the `WITH CHECK` fails, PostgreSQL raises a row-level security violation, and the mutation falls into `onError`.

```sql
-- CURRENT (broken for submission):
WITH CHECK (
  (SELECT public.is_org_member(org_id))
  AND status = 'draft'   -- ← blocks draft→submitted transition
);

-- FIX option A — remove WITH CHECK restriction:
WITH CHECK (
  (SELECT public.is_org_member(org_id))
);

-- FIX option B — move submission to Edge Function with service_role
```

**Impact:** The entire submission flow (SAVE-03 → SAVE-05 → SAVE-06 → history page) is unreachable for company users.
